Installation MeshCore – Guide rapide (FR)

Quelques minutes suffisent pour déployer vos premiers nœuds en France.

1

Matériel requis

  • 1 à n cartes LoRa compatibles MeshCore (ex. modules LoRa avec MCU intégré).
  • Un câble USB pour le flash.
  • Un smartphone Android ou iOS avec l’application MeshCore.
2

Flasher le firmware

  1. Placez la carte en mode bootloader (selon la carte : bouton au branchement, double‑reset, etc.).
  2. Ouvrez le Webflasher MeshCore (lien « Webflasher & App ») dans votre navigateur.
  3. Sélectionnez votre modèle de carte et le firmware adapté : Companion pour commencer.
  4. Lancez le flash, attendez la fin, puis débranchez/rebranchez la carte.
3

Appairage avec l’appli

  1. Activez le Bluetooth sur votre téléphone.
  2. Ouvrez l’appli MeshCore et ajoutez un nouvel appareil.
  3. Sélectionnez votre nœud, nommez‑le (ex. « FR75-MonPseudo-Toit‑Est ») et vérifiez qu’il apparaît en ligne.
4

Réglages LoRa France

Dans l’appli ou l’interface Web MeshCore, configurez :

  • Région : EU/UK (Narrow)
  • Fréquence : 869.618 MHz
  • Bande passante : 62.5 kHz
  • Spreading Factor : SF 8
  • Coding Rate : CR 8

Appliquez exactement ces réglages sur tous vos nœuds MeshCore France.

5

Premier mini‑réseau

  1. Placez deux nœuds à quelques dizaines de mètres, avec peu d’obstacles.
  2. Envoyez un message texte depuis l’appli reliée au premier nœud vers le second.
  3. Ajoutez un troisième nœud au milieu (Répéteur et non Compagnon) pour tester le routage multi‑sauts.
6

Aller plus loin

  • Déployez des répéteurs sur les hauteurs (toits, collines, tours).
  • Alimentez des nœuds sur batterie ou panneau solaire pour une couverture permanente.
  • Rejoignez le Discord France pour affiner vos réglages et partager.

Configurer un répéteur MeshCore

Un bon réglage de répéteur améliore fortement la portée et la qualité du maillage.

Principes importants

  • Ne pas mettre de mot de passe « guest » : donnez accès public à la télémétrie.
  • Activer la synchronisation d’horloge (clock sync) pour de meilleurs timings radio et un réseau plus stable.
  • Positionner le répéteur en hauteur (toit, colline, pylône) et le laisser alimenté en continu, avec une alimentation redondante si possible.
  • Installer une antenne performante, accordée à la bonne fréquence avec un filtre à cavité si possible.

Commandes recommandées

Depuis la console ou l’interface de configuration MeshCore du répéteur, appliquer :

set advert.interval 120
set flood.advert.interval 12
set agc.reset.interval 4
set multi.acks 1
set dutycycle 10
set path.hash.mode 1
set loop.detect minimal
set repeat on
set guest.password ''
set lon {longitude}
set lat {latitude}
  • advert.interval 120 : annonces zero-hop toutes les deux heures pour que la liste des voisins soit à jour.
  • flood.advert.interval 12 : garde une découverte de réseau régulière sans surcharger le canal.
  • agc.reset.interval 4 : réinitialise périodiquement le contrôle de gain automatique - très utile en milieu perturbé.
  • multi.acks 1 : active les accusés de réception multiples pour plus de fiabilité.
  • dutycycle 10 : Respect de la réglementation européene sur le duty-cycle.
  • path.hash.mode 1 : active le mode 2 octets pour les adresses.
  • loop.detect minimal : indique que l'on bloque les trames qui ont 4 fois le même id.
  • repeat on : indique que ce nœud est bien un répéteur du réseau.
  • guest.password '' : Permet l'accès sans mot de passe aux statistiques réseau.
  • lon / lat : renseigne la position GPS du répéteur pour les cartes et l’analyse réseau.

MeshCore vs Meshtastic

Deux approches LoRa mesh complémentaires, avec des priorités différentes.

Dimension MeshCore Meshtastic
Philosophie Optimisé pour des réseaux planifiés avec répéteurs dédiés, grande stabilité et contrôle du routage. Orienté “plug & play” pour groupes mobiles (rando, ski, vélo, camping).
Routage Routage par chemins appris et configurables, avec flood utilisé surtout pour la découverte et le secours. Routage par flood géré, où tous les nœuds rebroadcastent par défaut. Les les versions récentes de micro-logiciel permettent un controle plus fin via des roles.
Nœuds qui relaient Répéteurs dédiés pour porter le trafic exclusivement. Les compagnons (usage utilisateur) ne relaient pas pour réduire le bruit radio. Les clients relaient par défaut, ce qui simplifie les petits réseaux mais augmente le trafic sur les canaux.
Échelle du réseau Pensé pour des maillages plus grands et structurés, avec réglages fins (hops élevés, chemins définis). Très adapté aux réseaux de taille petite, avec une topologie plus dynamique et spontanée.
Licence Open Source (MIT) pour le firmware, les clients python, web et home-assistant mais propriétaire pour les applications mobile (Freemium). Open Source (GPLv3)
Réglages radio Profils orientés stabilité, airtime maîtrisé et fiabilité du transport de message. Profils variés moins centrés sur l’optimisation fine du trafic.
Cas d’usage typiques Infra locale semi‑fixe, relais sur toits/cols, réseaux de secours, maillages urbains ou régionaux. Groupes en mouvement, sorties outdoor, petites communautés, expérimentations rapides.

En pratique, beaucoup d’utilisateurs commencent avec Meshtastic pour se faire la main, puis déploient MeshCore lorsqu’ils veulent un réseau plus structuré qui fonctione de manière plus fiable.

Profil radio France – Vue rapide

Récapitulatif des paramètres LoRa recommandés pour le maillage MeshCore en France.

Paramètre Valeur France Commentaire
Région EU/UK (EU868) Bande libre européenne 863–870 MHz, usage LoRa.
Fréquence 869.618 MHz Canal dédié MeshCore France pour une meilleure interopérabilité.
Bande passante 62.5 kHz Narrowband : portée améliorée et robustesse accrue.
Spreading Factor SF 8 Bon compromis portée / débit pour messages texte.
Coding Rate CR 8 Correction d’erreur renforcée pour environnements bruyants.

Pour rester conforme à la réglementation EU868, pensez à respecter la puissance maximale autorisée (500mW de PIRE, soit <27dBm) et les limites de duty‑cycle (10%). Privilégiez le placement d'antenne (en hauteur) à la puissance.
